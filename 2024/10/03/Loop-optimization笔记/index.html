Static data dependence

a and a’ be two static array accesses.存在从a到a’的数据依赖，当且仅当

（1）a,a’有一个是写操作。

（2）存在动态实例o,o’符合上面2，3条件

Definition: a dependence is loop-carried if it crosses an iteration boundary

循环可以并行执行如果没有loop-carried dependence.

Data Dependence Analysis

只使用线性函数的循环边界和数组索引来做数据依赖分析。忽视非线性函数。

Replace a ≠ b with two sub-problems: a>b or a

Replace a =b with two sub-problems: a≤b and b≤a

GCD test, Banerjee’s test

2 outcomes

No → no dependence

Don’t know → assume there is a solution → dependence (Conservative)

GCD test

Theorem：线性丢番图方程a1x1+ a2x2 +…+ anxn = c 有实数解当且仅当gcd(a1 , a2 , …, an )能除c。

Banerjee’s Test

考虑循环上下界

Limitation：

1. 只考虑不等式有没有实数解而非整数解。因此有时有依赖不代表依赖存在。

2. 多下标数组会分开计算每个下标。这是conservative的，因为可能分开有解合起来没解。

3. 若loop limits和coefficients是未知的，则假定依赖存在。

求解不等式

Fourier-Motzkin Elimination

Affine Transform Theory

1. Iteration Space :A d-deep loop nest has d index variables, and is modeled by a ddimensional space. eg: i = 0,1,…….9

矩阵形式的迭代空间：

小小理解一下。

2. Data Space: bounded by array bounds.

矩阵形式的数组仿射访问：

3. Processor Space

先假定有无数个虚拟处理器，做完并行化后映射到物理处理器上。

−(vp1, p1), (vp2, p2), (vp3, p1), (vp4, p2),…

通过一个affine function将iteration space映射到processor space。

通过一个affine function还能从一个iteration space到另一个。在做interchange loops（循环置换）时使用。

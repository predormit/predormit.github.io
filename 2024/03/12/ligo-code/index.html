configargparse是一个Python库，它提供了一个命令行参数解析器，该解析器是argparse模块的扩展。它允许开发者从命令行、环境变量或配置文件中解析参数，使得程序的配置更加灵活和方便。

parser.add_argument()用来配置文件参数或向解析器添加命令行参数。

parser.add_argument('--config', is_config_file=True, help='Config file path')这行代码添加了一个命令行参数–config，用于指定配置文件的路径。is_config_file=True告诉argparse该参数是一个配置文件，help参数提供了该命令行选项的帮助信息。

parser.add_argument("--train_data_file", default=None, type=str, help="The input training data file (a text file).")这行代码定义了一个命令行参数–train_data_file，它接受一个字符串类型的文件路径作为训练数据文件的位置。default=None意味着如果没有指定这个参数，它的默认值是None。help参数提供了该命令行选项的帮助信息。

parser.add_argument("--do_train", action="store_true", help="Whether to run training.")这行代码添加了一个标志参数–do_train。action=”store_true”意味着如果指定了这个参数，它将被设置为True，否则为False。help参数解释了这个标志的作用，即指示程序是否执行训练。

parser.add_argument("--shuffle", action="store_true", help="Shuffle the training dataset")这个命令行参数–shuffle也是一个标志参数，它指示程序是否应该在训练之前打乱训练数据集。

parser.add_argument("--block_size", default=-1, type=int, help="Optional input sequence length after tokenization. ...")这行代码定义了一个名为–block_size的命令行参数，它接受一个整数类型的值。default=-1表示如果没有指定这个参数，它的默认值是-1。help字符串详细说明了这个参数的用途，即在分词后可选的输入序列长度。如果没有指定，训练数据集将根据模型的最大输入长度进行截断。

parser.add_argument("--logging_steps", type=int, default=500, help="Log every X updates steps.")这个参数名为–logging_steps，它接受一个整数类型的值，用于设置日志记录的频率。默认值是500，意味着每更新500步记录一次日志。

parser.add_argument("--ckpt_steps", type=int, default=1000, help="Checkpoint every X updates steps.")参数–ckpt_steps同样接受一个整数，用于设置模型检查点的保存频率。默认值是1000，意味着每更新1000步保存一次检查点。

parser.add_argument("--grow_scheme", default='none', type=str,choices=['none', 'ligo'],help="Method to grow the model: [none, ligo]")choices写法

--seed：这个参数接受一个整数，用于设置随机种子，以确保实验的可重复性，默认值为95。--per_gpu_train_batch_size：这个参数接受一个整数，用于设置每个GPU/CPU在训练时的批次大小，默认值为4。--per_gpu_eval_batch_size：这个参数同样接受一个整数，用于设置每个GPU/CPU在评估时的批次大小，默认值也为4。--gradient_accumulation_steps：这个参数接受一个整数，用于设置在执行反向传播和参数更新之前累积的更新步骤数，默认值为1。--max_steps：这个参数接受一个整数，用于设置要执行的总训练步数。如果设置了这个参数，它将覆盖num_train_epochs参数。

lamb优化器：优化器是用来更新和计算影响模型训练和模型输出的网络参数，使其逼近或达到最优值，从而最小化(或最大化)损失函数。 LAMB 出现的目的是加速预训练进程，这个优化器也成为 NLP 社区为泛机器学习领域做出的一大贡献。在使用 Adam 和 AdamW 等优化器时，一大问题在于 batch size 存在一定的隐式上限，一旦突破这个上限，梯度更新极端的取值会导致自适应学习率调整后极为困难的收敛，从而无法享受增加的 batch size 带来的提速增益。LAMB 优化器的作用便在于使模型在进行大批量数据训练时，能够维持梯度更新的精度。具体来说，LAMB 优化器支持自适应元素级更新（adaptive element-wise updating）和准确的逐层修正（layer-wise correction）。

线性预热(linear warmup)：warmup是针对学习率learning rate优化的一种策略，主要过程是，在预热期间，学习率从0线性（也可非线性）增加到优化器中的初始预设lr，之后使其学习率从优化器中的初始lr线性降低到0。由于刚开始训练时,模型的权重(weights)是随机初始化的，此时若选择一个较大的学习率,可能带来模型的不稳定(振荡)，选择Warmup预热学习率的方式，可以使得开始训练的几个epoch或者一些step内学习率较小,在预热的小学习率下，模型可以慢慢趋于稳定,等模型相对稳定后再选择预先设置的学习率进行训练,使得模型收敛速度变得更快，模型效果更佳。

权重衰减：正则化。

梯度裁剪：控制梯度范围，设定阈值。

scheduler(学习率调度器)：包括linear（线性衰减）、cosine（余弦退火）和poly（多项式衰减）

 if hasattr(self, 'get_depth_coeffs'):

        weight = torch.sum(self.get_weights() * coeffs_weights, -1)
        bias = torch.sum(self.get_bias() * coeffs_bias, -1)

    else:
        weight = self.get_weights()
        bias = self.get_bias()

    out_dim_expand = self.get_width_coeffs()
    if out_dim_expand is not None:
        weight = torch.cat([weight, torch.matmul(out_dim_expand, weight)], 0) # expand out dimension
        bias = torch.cat([bias, torch.matmul(out_dim_expand, bias)], 0) # expand out dimension

    if self.residual:
        weight = weight + self.residual_weight
        bias = bias + self.residual_bias

    return weight, bias

DistributedSampler确保每个进程都能访问到数据集的不同部分，从而实现数据并行化和加速训练。通过将数据集划分给多个进程，并在每个进程中使用采样器，可以在多个进程之间并行地处理数据，提高训练效率。在具体使用时，通常将 train_sampler 传递给数据加载器（例如 DataLoader），以便在训练过程中按照采样器的规则获取数据。:dataset： 用于采样的数据集。num_replicas（int，可选）： 参与分布式训练的进程的数量。默认情况下，:attr:world_size会从当前的分布式组中获取。rank (int, optional)： 当前进程在 :attr:num_replicas中的排名。默认情况下，:attr:rank会从当前的分布式组中检索。shuffle (bool, optional)： 如果True(默认)，采样器将洗牌索引。seed (int, optional): 如果 :attr:shuffle=True，随机的种子用于洗牌采样器。这个数字在分布式组的所有进程中应该是相同的。默认值:0.

        11. 2.1.2.11. lamb优化器：优化器是用来更新和计算影响模型训练和模型输出的网络参数，使其逼近或达到最优值，从而最小化(或最大化)损失函数。 LAMB 出现的目的是加速预训练进程，这个优化器也成为 NLP 社区为泛机器学习领域做出的一大贡献。在使用 Adam 和 AdamW 等优化器时，一大问题在于 batch size 存在一定的隐式上限，一旦突破这个上限，梯度更新极端的取值会导致自适应学习率调整后极为困难的收敛，从而无法享受增加的 batch size 带来的提速增益。LAMB 优化器的作用便在于使模型在进行大批量数据训练时，能够维持梯度更新的精度。具体来说，LAMB 优化器支持自适应元素级更新（adaptive element-wise updating）和准确的逐层修正（layer-wise correction）。
        12. 2.1.2.12. 线性预热(linear warmup)：warmup是针对学习率learning rate优化的一种策略，主要过程是，在预热期间，学习率从0线性（也可非线性）增加到优化器中的初始预设lr，之后使其学习率从优化器中的初始lr线性降低到0。由于刚开始训练时,模型的权重(weights)是随机初始化的，此时若选择一个较大的学习率,可能带来模型的不稳定(振荡)，选择Warmup预热学习率的方式，可以使得开始训练的几个epoch或者一些step内学习率较小,在预热的小学习率下，模型可以慢慢趋于稳定,等模型相对稳定后再选择预先设置的学习率进行训练,使得模型收敛速度变得更快，模型效果更佳。

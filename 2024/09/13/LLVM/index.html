LLVM

LLVM编译器分为多个模块，使用IR作为数据结构进行代码优化和生成。是基于SSA的表达。

项目组成

（1）Clang：解析 C/C++ 代码；（编译器）

（2）MLIR：构建可重用和可扩展编译器基础设施的新颖方法；（3）OpenMP：提供了一个OpenMP运行时库函数；

（4）polly：使用多面体模型实现了一套缓存局部性优化以及自动并行和向量化；

（5）LLDB（调试器）、 libc++（c++标准库） 、libc++ ABI（c++标准库ABI） 、compiler-rt、libclc、klee、LLD（链接器）.

（6）LLVM ：总的框架和工具链

(7) bolt: 二进制优化

allocainfo

  1. **DefiningBlocks**：保存 alloca 指令写入（store 指令）发生的基本块。这些块中存在给该 alloca 分配新值的操作。

  2. **UsingBlocks**：保存读取 alloca 值的基本块，即发生 load 操作的块。

  3. **OnlyStore**：如果有且仅有一个 store 操作，它会被存储在此处，表示该 alloca 的唯一 store 指令。如果有多个 store 指令，这个指针就不会使用。

  4. **OnlyBlock**：如果所有 store 和 load 操作都发生在同一个基本块中，该变量保存这个块的指针。如果涉及多个块

    def store . use load

LargeBlockInfo 类用于管理和维护在一个基本块中对 alloca 的加载和存储指令的相对顺序。它的设计目标是避免在处理同一基本块中的多个 alloca 时重复扫描整个基本块，从而提高效率。

1. **循环遍历 NewPhiNodes**：

  • 代码首先遍历 NewPhiNodes，这是一个 DenseMap，将某个键值对（std::pair<unsigned, unsigned>）映射到 PHI 节点 (PHINode *)。
  • I 是当前迭代器，E 是映射的终止位置。

2. 获取当前 PHI 节点：

  • SomePHI 是映射中当前遍历到的 PHI 节点 (PHINode *)。

3. 获取 PHI 节点所属的基本块：

  • SomePHI->getParent() 获取当前 PHI 节点所属的基本块 (BasicBlock *)。

4. 确保只处理基本块的第一个 PHI 节点：

  • 该代码的目标是只对每个基本块进行一次处理。因此，代码通过检查 BB->front() 来判断当前 PHI 节点是否是该基本块的第一个指令。如果 SomePHI 不是基本块中的第一个指令（即不是 BB->front()），那么这次循环将跳过当前 PHI 节点。
  • 这样做的目的是避免对同一基本块的多个 PHI 节点进行重复处理，因为所有 PHI 节点都在一个基本块的开头，处理一个即可。

这段代码的核心是从 PHI 节点的前驱块列表中移除已经提供输入的前驱块。通过先对 Preds 列表进行排序，然后使用二分查找（lower_bound）找到并移除 PHI 节点已经存在的前驱块。剩下的 Preds 列表包含那些没有为 PHI 节点提供输入值的前驱块，后续可能会对这些块进行处理，比如插入“毒值”（poison value）来补全缺失的输入

Csapp lab2 bomblab

解题思路

phase 1

调用了string not equal函数之前肯定有输入的string，所以往前找string.
去看%rsi里的地址，给了个#4150. x/s去找，发现是Why make trillions when we could make… billions?，就这样。

phase 2

有个函数叫read six numbers,所以phase 2的密码是6个数。从266d可以看出第一个数只能是0.然后又是一个循环，每次加上循环次数与下一个数相等。因此是0 1 3 6 10 15

phase 3

第三题可以看出，有很多行跳转到同一地方的指令，姑且认为它是一个switch语句。
然后再访问前面的地址，发现是%d %d，所以答案是两个数字。前面还有个限制就是说第一个参数是要小于7的。接着就是我这个炸弹最有意思的一点，switch的每一个语句最后都是mov 0 与我的第二个数作比较，所以我输出了随机一个小于7的数。4 0 就对了。

phase 4

phase 4是一个递归

fun4(rsp,8){

edi = 7;

eax = fun4(rsp,7);

r12d = eax + rsp;

edi = 6;

esi = rsp;

eax = fun4(rsp,6)

return fun4(rsp,6) + fun4(rsp,7) + rsp;

如图，同时rsp <= 4;得答案3，162；
但不知道为什么，答案是反的，162，3，折磨了我一天。

phase 5

字符串

用gdb x/s把#41e0，#41ae看一下，
发现是devils，就对应到每个字符在另一个字符串中的位置，拿着这个位置去字母表里对应的找，就找到了。
而且得到的一串字符串只有前15是有用的，因为有一行与0xf做and.

phase 6

phase 6有点长说是，中间还有一大段mov，看起来有点折磨。但最重要的就是看明白那个node。结果是要输出1到6的排列。它按照顺序将其存到了6个节点中。

因此，我们只需对node1到node6中的值排序就可以得出答案。node6还和node1到node5不在一个地方。
所以得出3 6 4 2 1 5
